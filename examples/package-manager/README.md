# Package manager walkthrough (DEC-316)

A runnable demo of `phg add` / `phg install` — how a phorj app declares and vendors a third-party
dependency. This directory is a complete, working project; the byte-identity differential runs
`src/main.phg` (importing the vendored dependency) on the interpreter, the VM, and transpiled PHP.

## Layout

```
package-manager/
  phorj.json               # the manifest (composer.json-style)
  phorj.lock               # the pinned, integrity-hashed resolution (generated)
  src/main.phg             # the app entry — imports Acme.Greet
  greet-src/greet.phg      # the dependency's SOURCE (what a publisher writes)
  vendor/Acme/Greet/       # the INSTALLED dependency (generated by `phg install`)
```

`vendor/` and `phorj.lock` are the *output* of `phg install` — committed here so the demo is runnable
out of the box (and gated). In a real project you'd `.gitignore vendor/` and commit only
`phorj.json` + `phorj.lock`.

## The manifest — `phorj.json`

Composer-style: a `name`, a `version`, and a `require` map of `Publisher/Name` → source. Three source
kinds are supported (this demo uses a local **path** so it needs no network):

```json
{
  "name": "Acme/DemoApp",
  "version": "0.1.0",
  "require": {
    "Acme/Greet": { "path": "greet-src" },
    "Acme/Json":  "^1.2",
    "Foo/Bar":    { "git": "https://example.test/foo/bar.git", "ref": "v2.1.0" }
  }
}
```

- `"^1.2"` (a bare string) → the **registry** (resolved via the registry index by semver constraint).
- `{ "git": …, "ref": … }` → a **git** repo at a tag/branch/commit.
- `{ "path": … }` → a **local** directory (dev dependency).

## The commands

```console
$ phg install                       # resolve require -> fetch -> vendor/ -> write phorj.lock
Installed 1 package(s):
  Acme/Greet path

$ phg run src/main.phg
Hello from the Acme.Greet package!
```

Other verbs:

```console
$ phg add Acme/Json@^1.2            # registry dependency
$ phg add Foo/Bar --git https://example.test/foo/bar.git --ref v2.1.0
$ phg add Dev/Local --path ../local
$ phg update                        # re-resolve to the newest satisfying versions
$ phg remove Acme/Greet             # drop from phorj.json + vendor/, re-resolve
```

`phg add/install/update/remove` are the **only** network-capable commands — `phg run`/`check`/
`transpile` never touch the network (Invariant 10). Every install pins each package with a tree
SHA-256 in `phorj.lock`; a later `phg install` re-verifies those hashes offline and refuses a
tampered or stale `vendor/` tree.

## How resolution works

`import Acme.Greet;` resolves through the DEC-282 loader's third search root,
`vendor/<Publisher>/<Name>/` (folder = package, `Publisher.Name.*` namespace, `Core.*` reserved for
first-party). The package is pure `.phg` source (DEC-315), so it transpiles to PHP exactly like your
own code — which is why this example passes the byte-identity spine with no special handling.
