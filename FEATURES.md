# Features

A capability matrix for Phorj ‚Äî what works **today** versus what is **planned**. For runnable proof
of the "today" column, see [`examples/`](examples/README.md); for the forward plan see
[ROADMAP.md](ROADMAP.md); for things that are deliberately rejected-but-clean, see
[KNOWN_ISSUES.md](KNOWN_ISSUES.md).

## Project qualities

- **Std-first with a short, vetted, feature-gated dependency list** ‚Äî `argon2` (Argon2id),
  `regex` (`Core.Regex`), `ctrlc` (signals), `corosensei` (green threads), `rusqlite`/`postgres`/

  HttpClient/Mail); nothing else (see [THIRD-PARTY-NOTICES.md](THIRD-PARTY-NOTICES.md) and
  `docs/specs/UNIFIED-SPEC.md#external-dependency-policy` ‚Äî every admission is register-recorded).
- **No `unsafe` outside one audited island** ‚Äî `#![deny(unsafe_code)]` on both crate roots; the
  JIT's audited `unsafe` (confined to `src/jit/`) is the sole exception.
- **Never panics on input** ‚Äî adversarial source *and* adversarial binaries are handled cleanly
  (invariant EV-7).
- **Differential-tested** ‚Äî every example runs on both backends and must match byte-for-byte.
