## 1. Ladder placement (invariant 14) — case 2, native-only

`Core.Mail` is **native-only**. Transpile is a **hard error `E-TRANSPILE-MAIL`**. PHP's stdlib `mail()`
has no SMTP auth, no TLS, and is header-injection-prone — there is no faithful safe PHP map, and any
attempt (e.g. text-only → `mail()`) would silently drop auth/TLS/attachments (a rule-14-forbidden
downgrade). The mailer joins the `E-TRANSPILE-*` / native-only list (concurrency `E-CONCURRENCY-NO-PHP`,
`E-TRANSPILE-UNCHECKED`, `E-TRANSPILE-MONGO`).

**Spine quarantine:** the natives are `pure:false`, so `uses_impure_native` auto-excludes any program
using `Core.Mail` from `differential.rs` (network + side effects can't be byte-identity-gated).
Validated instead by `tests/mail.rs` against the stack's Mailpit faker + deterministic `file`/`null`
transports — exactly the Core.Db `tests/db.rs` pattern.

- Add `E-TRANSPILE-MAIL` raised in `src/transpile/` (mirror `E-TRANSPILE-UNCHECKED` in
  `transpile/functions.rs`), a `cli/explain.rs` entry, and a disclosure paragraph wherever byte-identity
  is claimed.
- Register the impure module in `process_tests.rs` `impure_modules` (like `Core.DbSys`).

## 2. Module split (nothing in the wind)

Mirror Core.Db's two-module split:
- **`Core.Mail`** — the prelude: `Mailer`, `Email`, `Address`, `Attachment`, `MailError`, `SmtpConfig`,
  the transport constructors, and the prelude-local `MailResult<T>` Ok|Err enum. User writes
  `import Core.Mail;` (+ `import Core.Mail.MailError;` etc. for bare type use).
- **`Core.MailSys`** — the native-only runtime natives (the qualifier is disjoint from the prelude
  classes, as `Core.DbSys` is disjoint from `class Db`). Reserved handle type `MailHandle`
  (import-gated on `Core.MailSys`), like `DbHandle`.

Error mechanism = the **prelude-wrapper** exactly as Core.Db: natives return the prelude-local
`MailResult<T>` (Ok/Err) — NOT `Core.Result` (importer-after-imported injection ordering forbids it);
the prelude matches and throws a catchable `MailError` via a `never`-returning `MailError.fail` helper
(throw is a statement, not an expression).

## 3. Transports (behind a `MailTransport` trait — mirrors the Db driver trait)

Ship all four in v1 (developer ruling):

| Transport | Constructor | Notes |
|---|---|---|
| **SMTP** (optional auth) | `new Mailer(new SmtpConfig(host, port))` (no-auth) · `new Mailer(new SmtpConfig(host, port, user, Secret pw))` (authed) | Auth is **optional** — Mailpit/MailHog fakers accept no-credential connections. TLS-by-default (STARTTLS; implicit-TLS configurable). |
| **sendmail** | `new Mailer(new SendmailTransport())` (default `/usr/sbin/sendmail`, path overridable) | Local MTA pipe. |
| **file** | `new Mailer(new FileTransport(dir))` | Writes each message as `.eml` to `dir` — deterministic offline test transport. |
| **null** | `new Mailer(new NullTransport())` | Discards (dry-run / test). |

`Secret` = the SAME credential type as the Core.Db driver slice G (redacted in every error/log/toString).
The SMTP password must be a `Secret` so a connect error prints host but never the password.

## 4. Composition surface (full rich — developer ruling)

Instance-based, mandatory `new`, SOLID (global tenets). Builder shape:

```phorj
import Core.Mail;

Mailer m = new Mailer(new SmtpConfig("localhost", 1025)); // Mailpit, no auth
Email e = new Email()
  .from(new Address("app@x.io", "App"))
  .to(new Address("user@y.io"))
  .cc(new Address("obs@y.io"))
  .replyTo(new Address("noreply@x.io"))
  .subject("Welcome")
  .html("<h1>Hi</h1><img src=\"cid:logo\">")   // auto-derives a plaintext alternative
  .attachInline("logo", Attachment.fromFile("logo.png"))
  .attach(Attachment.fromFile("invoice.pdf"));
m.send(e);   // throws MailError on failure
```

- **`Address`** — typed, injection-safe by construction (no raw-header injection possible — the #1 PHP
  `mail()` footgun). Validate on construct → `E-MAIL-INVALID-ADDRESS` (or a runtime `MailError` subtype;
  pick the Core.Db-consistent choice). Optional display name. IDN/punycode handled by lettre.
- **`.html(body)`** — sets an HTML part and **auto-derives a plaintext alternative** (`multipart/alternative`).
  `.text(body)` overrides the auto-plaintext if the user supplies their own.
- **`.attachInline(cid, Attachment)`** — inline CID image (referenced as `cid:<cid>` in the HTML).
- **`.attach(Attachment)`** — file attachment. `Attachment.fromFile(path)` / `Attachment.fromBytes(name, mime, bytes)`.
- Multi-recipient: `.to`/`.cc`/`.bcc` accept repeated calls (accumulate) — decide list-vs-repeat to match
  the Core.Db builder ergonomics.
- **Batch:** `m.sendAll(List<Email>)` over a reused connection (primitive, NOT a queue — queuing/retry-
  scheduling/templating is userland, same primitive-vs-ORM line as Core.Db).

## 5. Typed error taxonomy (`MailError` — shaped like `DbError`)

Subtypes (map from lettre's error kinds): `ConnectionFailed`, `AuthFailed`, `RecipientRejected`,
`TlsError`, `InvalidAddress`, `MessageBuildFailed`, `Timeout`, `Io`, `Other`. Same prelude match +
`never`-returning `.fail` mechanism as `DbError`. Redact any `Secret` in messages.

## 6. Dependency — `lettre` (ADMITTED, DEC-223)

```toml
# Cargo.toml, non-wasm target block (like rusqlite/postgres)
mail = ["dep:lettre"]
lettre = { version = "0.11", default-features = false,
           features = ["smtp-transport", "rustls-tls", "builder", "dkim"] }
```

Rationale: mature de-facto standard, RFC-correct MIME/multipart/attachments, SMTP auth,
STARTTLS/implicit TLS via already-admitted **rustls**, optional **DKIM** signing, and a **blocking
`SmtpTransport`** so it stays **no-tokio**. Rejected alternatives: `mail-send`+`mail-builder` (Stalwart;
modern + DKIM built-in but **tokio-async** → violates no-tokio); hand-roll SMTP+MIME over std+rustls
(large RFC/MIME/encoding bug surface lettre already gets right).

- Feature `mail`: **non-default, non-wasm-gated** (like `db`). Verify `cargo check --no-default-features`
  still compiles (mail off) and wasm build excludes it.
- Optional convenience: a `db-all`-style umbrella is NOT required; keep `mail` standalone.
