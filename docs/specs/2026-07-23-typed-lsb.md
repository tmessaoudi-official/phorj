# SPEC — Typed LSB: the `Self` return type (DEC-331 D10c design slice)

> Status: **SPEC FROZEN, awaiting dev ruling.** D10c ruling: base static/factory resolves to
> the called subclass — WITHOUT PHP's `self::`/`static::`/`$this::`/`parent::` four-way
> confusion. One keyword, statically typed.

## 1. Surface

```phg
class Model {
    static function create(): Self { return new Self(); }   // Self = the CALLED class
    function copy(): Self { /* ... */ }
}
class User extends Model {}

User u = User.create();   // : User — NOT Model (the LSB win, statically typed)
User v = u.copy();        // : User
```

## 2. Semantics

- `Self` is a TYPE (return positions + `new Self()` construction) meaning "the class the
  call was dispatched on" — PHP's `static` (late static binding), but resolved in the TYPE
  SYSTEM: at a call site `X.create()` / `x.copy()`, the checker substitutes `Self := X`
  (the static receiver type). No runtime type juggling.
- Soundness rule (the classic LSB pitfall, checked at COMPILE time): a class inheriting a
  `new Self()`-containing method must satisfy its construction — v1 takes the strict route:
  `new Self()` is legal only when every subclass keeps a compatible constructor signature
  (`E-SELF-CTOR-MISMATCH` on the subclass otherwise). This is what PHP cannot check and
  where `new static()` blows up at runtime.
- `Self` in parameter position: REJECTED in v1 (contravariance trap — `E-SELF-PARAM`);
  return + construction only. Field type `Self`: rejected v1 (`E-SELF-FIELD`).
- Interplay with overriding: an override may narrow `Self`-typed returns naturally (they
  already mean the receiver); annotating the override with the concrete class is also legal.

## 3. Backends (Invariant 17)

- **Checker**: `Self` resolves at each call site — pure type-level; ERASED before backends
  (Invariant 5: backends never see `Self`). `new Self()` compiles to the existing
  late-bound-class construction the VM/interp already have for static dispatch (receiver
  class threaded through static calls — verify the existing static-call receiver plumbing at
  build; if absent, one `Op` extension → three exhaustive matches, Invariant 3).
- **Transpile**: `new Self()` → `new static()`; `: Self` return type → `: static` (PHP 8
  native, faithful tier 1).
- **Lift**: PHP `static`/`self` return types + `new static()`/`new self()` lift to `Self` /
  the literal class respectively — closes a lift gap.

## 5. PENDING for dev

- **P1**: keyword spelling — `Self` (recommended: matches PascalCase type naming SSOT) vs
  `static` (PHP-familiar but keyword-overloaded).
- **P2**: strict ctor rule (recommended) vs PHP-style permissive `new Self()` with a runtime
  fault — strict is the "better than php" position but constrains subclass constructors.
