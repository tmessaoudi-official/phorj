## 1. Connection & drivers
- **Multi-driver architecture** вЂ” `new Db(dsn)` dispatches on the DSN scheme behind a driver trait:
  `sqlite:вЂ¦` (rusqlite, shipped) В· `postgres://вЂ¦` В· `mysql://вЂ¦` (MariaDB = MySQL protocol). Each driver
  is a vetted, feature-gated, **sync** crate (`db-postgres`, `db-mysql`; async/tokio stays disallowed per
  the dependency policy), spine-quarantined + fixture-tested. Oracle deferred (no clean pure-Rust sync
  driver). Architecture is **expandable** вЂ” a new backend is one driver impl + one dep admission.
- **Throwing constructor** (DEC-221): `new Db(dsn) throws DbError` (fail-fast, PHP-`new PDO`-faithful).
- **Credential safety**: the password is a `Core.Secret` so it is **redacted in every error / log /
  toString** вЂ” a connect error prints host but never the password (PDO leaks the DSN in exceptions).
- **`using`/`Closable`** (DEC-203): `using (Db db = new Db(dsn)) { вЂ¦ }` closes the connection
  deterministically (RAII) even on early return/throw.

## 2. Statements & binding
- **Prepared-first**: `db.prepare(sql)` в†’ `Statement`; every path goes through a statement.
- **Both bind styles**: positional `.bind(v)` (`?`) and named `.bindNamed("n", v)` (`:name`), mutually
  exclusive per statement, chainable, typed (`string | int | float | bool | null | bytes`).
- **Typed `IN`-list binding**: `.bindList([1,2,3])` on `вЂ¦ IN (?)` auto-expands to `(?,?,?)` with each
  value bound вЂ” strictly better + safer than PDO (which can't bind arrays to `IN`).
- **SQL-injection compile-time lint**: a variable string-interpolated into a `prepare(...)` SQL literal
  (`prepare("вЂ¦ {userId}")`) is flagged (`W-SQL-INJECTION`) and steered to a bind вЂ” kills injection at
  compile time (PDO can't).
- **Compile-time placeholder/bind arity check** (literal SQL): `?` count vs `.bind()` count в†’ compile
  error on mismatch (best-effort for dynamic chains).

## 3. Results вЂ” hydration
- **Dynamic path** (no class): `query()` в†’ `List<Row>`, `r.getInt/getString/getFloat/getBool("col")`
  (+ nullable variants), typed + strict (no silent coercion). The no-class escape hatch.
- **Typed-generic path**: `queryInto<T>()` в†’ `List<T>`; `queryOneInto<T>()` в†’ `T?` (0в†’null, 1в†’obj,
  >1в†’`DbError`). T specified either **contextually** (`List<User> u = вЂ¦queryInto();`, shipped) OR via
  **turbofish** (`вЂ¦queryInto<User>();`) вЂ” turbofish is a general language feature to add (method type
  args). Mapping is **by field NAME, STRICT**: missing col / type mismatch / NULL-into-non-optional в†’
  `DbError`; `T?` fields admit NULL; extra columns ignored.
- **Flat OR nested вЂ” chosen by T's SHAPE** (one generic call, no flag): a **scalar** field в†’ its column
  (flat); an **entity** field в†’ recursively hydrated from prefix-aliased columns (nested), the alias
  convention being **dotted** `"order.total"` (quoted identifier); an **optional entity** field
  (`Order? order`) в†’ null when all its columns are NULL (LEFT JOIN). Flat is the zero-config default.
  **EAGER only** вЂ” nested is filled from the JOIN in ONE query (zero N+1). Lazy relation loading is
  REJECTED in the primitive (N+1, live-connection-bound stateful objects, ORM-territory) вЂ” it's a
  userland ORM concern.
- **`queryScalar<T>()`** вЂ” one typed value (`SELECT COUNT(*)`/`MAX`). **`queryMap<K,V>()`** вЂ” index rows
  into a `Map<K,V>` by a key column (idв†’entity).
- **Lazy streaming**: `streamInto<T>()` yields `T` row-by-row (constant memory for huge results) вЂ” the
  real "laziness" (result streaming), distinct from lazy relations (which we don't do).
- **Column naming strategy**: configurable snake_case-DB в†” camelCase-phorj mapping (`user_name`в†’
  `userName`) so you don't alias every column; strict-exact stays the default.
- **Value mapping**: DB value в†” phorj `enum` (a `status` column в†’ an enum, strict); numeric в†’
  `decimal` (exact **money**, never float); timestamp в†’ `DateTime` (gated by DEC-206); JSON/`jsonb` в†’
  a phorj type via `Core.Json`; Postgres array (`int[]`) в†’ `List<T>`.

## 4. Writes
- `exec()` в†’ affected-row count. **`lastInsertId()` / `RETURNING`** вЂ” the auto-generated PK after an
  INSERT (essential for autoincrement/serial keys). **Bulk `executeMany(rows)`** вЂ” one prepared
  statement, many bind-sets (far faster than a loop).

## 5. Transactions
- `db.transaction(() => { вЂ¦ })` вЂ” commit on normal return, **auto-rollback on throw**. **Savepoints**:
  a nested `transaction` becomes a savepoint (composable partial rollback). **Isolation levels**:
  `db.transaction(Isolation.Serializable, fn)` (ReadCommitted/RepeatableRead/Serializable). **Retry**:
  `db.transaction(retries: N, вЂ¦)` auto-retries transient `Deadlock`/`SerializationFailure` (Serializable
  is only practical with retry).

## 6. Errors вЂ” typed taxonomy
- Catchable `throws DbError` (never PDO's silent false/null). **Subtyped**: `UniqueViolation`,
  `ConstraintViolation`, `ConnectionError`, `Timeout`, `SerializationFailure`/`Deadlock`, `SyntaxError`
  (+ the underlying driver code). Callers `catch (UniqueViolation e)` vs `catch (ConnectionError e)` etc.
  Load-bearing: **retry** detects `Deadlock`/`SerializationFailure`; **timeout** produces `Timeout`.

## 7. Observability & robustness
- **`Db.onQuery((sql, ms) => вЂ¦)`** вЂ” per-query hook for logging / metrics / slow-query / tracing.
- **Query timeout**: `.timeout(ms)` в†’ a runaway query fails with `DbError.Timeout` instead of hanging.

## 8. Spine / LADDER
- LADDER case-1 (faithful в†’ PHP PDO for transpile). Execution is `pure:false` в†’ spine-quarantined from
  the byte-identity differential (live DB I/O в‰  byte-identical rusqlite-vs-PDO); `run в‰Ў runvm` holds
  (shared eval bodies); correctness via `tests/db.rs` fixtures on both Rust backends.

## Out of scope (userland or a separate wave вЂ” NOT the primitive)
- **Userland library (ORM/builder):** relations/lazy loading, unit-of-work, migrations, `upsert`/
  query-builder DSL, soft-deletes, optimistic locking.
- **Infra/userland:** connection pooling, read-replica routing, query-result caching.
- **Separate future wave:** sqlx-style **compile-time SQL-vs-schema checking** (needs a schema source).
- **Dialect-advanced, later:** Postgres `COPY`/pipeline/`LISTEN`/`NOTIFY`.
