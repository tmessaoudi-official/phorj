# M8.5 — Foreign PHP interop (`declare` / `.d.phg`) — design

Status: **design-locked** (2026-06-28). Milestone 2 of the "all four, in order" marathon.

## Goal

Let Phorge code call **existing PHP** (built-ins and library classes) with full type-checking, and
transpile to idiomatic PHP that uses them directly. This is the **migration bridge** — adopt Phorge
incrementally over a PHP codebase — and is explicitly a transpile-target feature.

## The spine decision (developer-endorsed at milestone selection)

Foreign PHP only exists in the PHP world; the Rust backends (interpreter/VM) have no PHP runtime, so a
foreign call cannot execute on them. Therefore a program that uses foreign declarations is
**PHP-target-only**, *quarantined from the `run ≡ runvm` byte-identity oracle*:

- **Pure Phorge stays 100% byte-identical** — the spine is untouched.
- A program containing any `declare` item is rejected by `run`/`runvm` with **one clean pre-flight
  message** (`E-FOREIGN-RUNTIME`: foreign symbols require the PHP target — use `phg transpile`). No
  per-call fault machinery in the execution paths — the gate is a single scan, zero backend surgery.
- `check` (type-checking) and `transpile` work fully. Validation of interop is by **type-checking +
  transpiled-PHP golden output** (a new `tests/interop.rs` harness runs the PHP leg only).

This honors the "every *Phorge feature* is native on the Rust backends; never delegate to PHP" rule:
interop is not a Phorge feature being implemented on PHP — it is a bridge to *external* code, which by
nature lives only on the PHP target. `Phorge : PHP :: TypeScript : JavaScript`, and `.d.phg : .d.ts`.

## Syntax

`declare` (contextual at item position, like `test`/`when`/`as` — not a reserved word) introduces a
**signature-only, bodyless** foreign symbol:

```
declare function strlen(string s) -> int;
declare function json_encode(mixed value) -> string;

declare class DateTimeImmutable {
  constructor(string datetime);
  function format(string fmt) -> string;
  static function createFromFormat(string fmt, string datetime) -> DateTimeImmutable;
}
```

A `declare` symbol produces **no PHP definition** (PHP already has it); references to it transpile to the
global PHP form (leading `\`): `\strlen($s)`, `new \DateTimeImmutable(...)`, `$d->format(...)`,
`\DateTimeImmutable::createFromFormat(...)`, field read `$o->prop`.

## Representation

A new `foreign: bool` on `FunctionDecl` and `ClassDecl` (default `false` everywhere; set `true` only on
the parser's `declare` path). Chosen over a new `Item::Foreign` variant — a bool field breaks only the
finite, compiler-enforced set of struct literals, not the many exhaustive `Item` matches (fmt, walk,
checker, transpiler, compiler, interpreter, loader, lift). Maximizes reuse: the checker registers a
foreign function/class signature through the *existing* collection pass; only foreign-specific behavior
(skip body checks, gate run/runvm, transpile to `\Name`) is new.

## Slices

- **S1 — `declare function`.** Parse; checker registers + validates calls (skip body); `run`/`runvm`
  pre-flight gate (`E-FOREIGN-RUNTIME`); transpiler emits `\name(args)` and skips the definition;
  `tests/interop.rs` (transpile→PHP golden, no run/runvm); `examples/interop/` companion + README
  (excluded from the `differential.rs` glob).
- **S2 — `declare class`.** Foreign class with constructor / methods / static methods / public fields /
  consts; `new \Name`, `->method`, `\Name::static`, `$o->field`. Type-checking against members;
  `instanceof` allowed.
- **S3 — `.d.phg` files + foreign exceptions.** Import a `*.d.phg` declaration file (a file of `declare`s)
  into a program; `catch` a foreign PHP exception (a `declare class … extends Error`-shaped bridge).
  File discovery may be deferred to a follow-up if large — inline `declare` covers the core.

## Verification

`check`/`transpile` exercised by `tests/interop.rs` golden against real PHP 8.5 (`php -n`); pure-Phorge
suites unchanged (spine untouched). `cargo clippy -D warnings`, `cargo fmt --check`.

## Deferred (KNOWN_ISSUES)

Running foreign-using programs on the Rust backends (needs a PHP FFI — out of scope); foreign *generic*
PHP types; namespaced foreign imports beyond a single leading `\`; automatic `.d.phg` generation from
PHP; Composer-package declaration bundling.
