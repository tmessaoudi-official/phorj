# M-mut.2 — compound-assign + `++`/`--` + `??=` Implementation Plan

> Slice 2 of the mutation milestone (`docs/specs/2026-06-21-mutation-milestone-design.md` §4).
> Autonomy: built without checkpoints per directive `f5f2912`.

**Goal:** Add `+= -= *= /= %=`, `??=`, and statement-form `x++`/`x--` as **pure parser+lexer
desugar** into the `Stmt::Assign` that M-mut.1 already wired — no new `Op`, no GC, no checker /
backend / transpiler / loader changes.

**Architecture:** Each compound form lowers in the parser:
- `x op= e`  ⟶  `x = x op e`   (`op` ∈ `+ - * / %`)
- `x ??= e`  ⟶  `x = x ?? e`
- `x++` / `x--` ⟶ `x = x + 1` / `x = x - 1`

`/=` and `%=` inherit the `__phorge_div`/`__phorge_rem` transpile routing of `/` and `%` for free
(F7) because they desugar to the existing `BinaryOp::Div`/`Rem`. Immutability / unknown-var / type
checks are inherited from the M-mut.1 `Stmt::Assign` checker arm. Statement form only (no prefix
`++x`, no expression-position increment) — matches spec §4 "`n++`/`n--` (stmt form)".

**Global constraints:** std-only Rust; spine `run ≡ runvm ≡ real PHP` byte-identical
(`PHORGE_REQUIRE_PHP=1`); clippy + fmt clean; examples ship with the feature.

---

## Task 1: lexer tokens

**Files:** `src/token.rs`, `src/lexer.rs`

- New `TokenKind`: `PlusEq MinusEq StarEq SlashEq PercentEq PlusPlus MinusMinus QuestionQuestionEq`.
- Two-char dispatch arms: `+= -= *= /= %= ++ --` (each distinct second byte; `->`/`--`/`-=` coexist).
- `??=` is **three-char**: add a longest-match block ahead of the two-char `??`, mirroring `..=`/`..`.
- RED: `multi_char_operators`-style test asserting the eight new tokens lex; verify it fails first.

## Task 2: parser desugar

**Files:** `src/parser.rs` `parse_var_decl_or_expr_stmt`

- After the existing plain-`=` reassignment branch: a `compound_op(&TokenKind) -> Option<BinaryOp>`
  helper maps the five `op=` tokens + `??=` (→ `Coalesce`) to a `BinaryOp`; build
  `Stmt::Assign { target: expr.clone(), value: Binary { op, lhs: expr.clone(), rhs }, span }`.
- `PlusPlus`/`MinusMinus`: no RHS — synthesize `Binary { Add|Sub, expr.clone(), Int(1) }`.
- Synthesized nodes carry the statement span `sp` (diagnostics still point at the real target).
- RED: parser tests asserting each form lowers to the expected `Stmt::Assign` shape; verify failure.

## Task 3: checker lock test (no impl change)

**Files:** `src/checker.rs` tests

- `x += 1` on an immutable `x` ⟶ `E-ASSIGN-IMMUTABLE` (inherited); `x ++` on unknown ⟶ `-UNKNOWN`.
- Confirms the desugar correctly reuses the M-mut.1 arm.

## Task 4: differential parity

**Files:** `tests/differential.rs`

- `mutation_compound_agrees()`: `+= -= *= /=` accumulators; `%=` with **negative dividend**
  (sign-follows-dividend, spec §8 #3); `??=` on an optional; `++`/`--` counter. All PHP-gated.

## Task 5: guide example + README

**Files:** `examples/guide/compound-assign.phg`, `examples/README.md`

- A `package Main` program exercising every form with a two-binding observe (value-copy) check;
  byte-identical on run/runvm/real PHP.

- `cargo test`, `PHORGE_REQUIRE_PHP=1 cargo test --test differential`, `cargo clippy --all-targets`,

- #3 `%=` negative operands: inherited from `%`/`__phorge_rem`, already byte-identical — add a
