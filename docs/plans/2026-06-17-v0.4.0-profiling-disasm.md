# v0.4.0 Quick Wins — Profiling + Disasm Plan

> Folded into the (locally tagged, unpushed) **v0.4.0** tag. Three pieces, one green commit each.

**Goal:** Add per-backend memory profiling to `phorge bench`, ship `phorge disasm`, and add a
profiling example — closing out v0.4.0 before publish.

**Architecture:** `bench` already does per-phase *timing* (parse+check / compile / tw run / vm run +
verdict, `src/cli.rs::bench_report`). The only new thing is **memory**: a std-only, Linux-only
`src/mem.rs` reads `/proc/self/status` (`VmRSS`=current, `VmHWM`=peak) and resets the peak via
`/proc/self/clear_refs`=5, so bench reports *baseline-independent per-phase peak growth*. Non-Linux →
graceful "memory: unavailable". `disasm` dumps the compiled `BytecodeProgram` using `Op`'s `Debug`
(no new match surface to drift) plus const/name annotations. The example is auto byte-identity-gated
because `tests/differential.rs` globs `examples/**/*.phg`.

**Tech stack:** Rust 2021, std-only, `#![forbid(unsafe_code)]`, `[lints] warnings="deny"`.

## Task 1 — `src/mem.rs` memory sampling (TDD)
- Create `src/mem.rs`: `current_rss_kb()`/`peak_rss_kb()` (parse `/proc/self/status`), `reset_peak_rss()` (write `5\n` to `/proc/self/clear_refs`, ignore error). All `Option`/no-op off-Linux.
- `pub mod mem;` in `src/lib.rs`.
- Test: on Linux `current_rss_kb()`/`peak_rss_kb()` are `Some`; `reset_peak_rss()` never panics anywhere.
- Commit: `feat(bench): /proc-based RSS sampling module (mem.rs)`.

## Task 2 — memory section in `bench` (TDD)
- `peak_growth_of(f)`: reset peak, sample current, run `f` once, return `peak - before` (saturating), `None` off-Linux.
- Extend `bench_report` with a memory block (front/compile/tw-run/vm-run peak growth + a tw-vs-vm verdict + overall current/peak RSS). `fmt_kb(Option<u64>)` renderer (`KiB`/`MiB`/`n/a`). All-`None` → "memory: unavailable on this platform".
- Test: `bench_report` output contains a `memory` header (always printed).
- Commit: `feat(bench): per-phase peak-RSS memory profiling`.

## Task 3 — `phorge disasm` (TDD)
- `cmd_disasm(src)`: parse_checked → compile → format. Per function: `fn #i name/arity:` then `ip  Lline  {op:?}  ; annotation`; annotate `Const` (const value), `Call` (fn name), `GetField`/`CallMethod`/`MakeEnum`/`MatchTag`/`MakeInstance` (resolved name); program header (nfns, main) + non-empty descriptor tables.
- Wire `"disasm"` into `main.rs` whitelist + dispatch; add to `help_text()` + `USAGE`.
- Test: disasm of `function main() { println(1 + 2); }` contains `AddI`, `Print`, `main`.
- Commit: `feat(cli): phorge disasm — bytecode disassembler`.
