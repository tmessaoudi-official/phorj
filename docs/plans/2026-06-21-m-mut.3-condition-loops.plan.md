# M-mut.3 — condition loops (`while`, do-while, C-`for`, while-let) + `break`/`continue`

> Slice 3 of the mutation milestone (`docs/specs/2026-06-21-mutation-milestone-design.md` §4).
> Autonomy: built without checkpoints per directive `f5f2912`. F5: **no new loop opcode** — every
> loop lowers to existing `Jump`/`JumpIfFalse`/`SetLocal`.

**Goal:** Condition-driven loops, made useful by M-mut.1/.2 mutation. `while (cond) {}`,
`do {} while (cond);`, C-style `for (init; cond; step) {}`, while-let `while (var x = opt) {}`, plus
`break;`/`continue;`. Byte-identical `run ≡ runvm ≡ real PHP`.

**Architecture:**
- **AST:** `Stmt::While { cond, bind: Option<String>, body, post_cond: bool, span }` folds three
  forms — plain `while` (`bind=None, post_cond=false`), `do-while` (`post_cond=true`), while-let
  (`bind=Some`). `Stmt::CFor { init: Option<Box<Stmt>>, cond: Option<Expr>, step: Option<Box<Stmt>>,
  body, span }`. `Stmt::Break(Span)` / `Stmt::Continue(Span)`.
- **Interpreter:** new `Signal::Break` / `Signal::Continue`; loop arms catch them (`Break` ⇒ stop,
  `Continue` ⇒ next iteration). The checker guarantees they never escape a loop.
- **VM (F5):** a compiler `loop_frames` stack — each frame holds `break_jumps`/`continue_jumps`
  backpatch vectors (the `ctor_return_jumps` model) + `body_base` (the `self.locals.len()` to which
  break/continue pop). `break`/`continue` emit `Op::Pop × (locals.len() - body_base)` then a
  placeholder `Jump`; patched to exit / continue-target at loop end. Loops are `JumpIfFalse`-guarded
  back-edges. Line-919 height re-anchoring makes the divergent jumps safe (same as dead-code-after-
  `return`).
- **Transpiler:** PHP `while`/`do…while`/`for`/`break`/`continue`; while-let lowers to
  `while (($x = expr) !== null)`.
- **Checker:** a loop-depth counter rejects `break`/`continue` outside a loop
  (`E-BREAK-OUTSIDE-LOOP`/`E-CONTINUE-OUTSIDE-LOOP`); while-let binds the non-null inner like if-let.

**Global constraints:** std-only; spine byte-identical (`PHORGE_REQUIRE_PHP=1`); clippy+fmt clean;
examples ship with the feature.

## Step group A — `while` / do-while / while-let + `break`/`continue` (commit 1)

1. Lexer: keywords `while`, `do`, `break`, `continue` (+ test).
2. AST: add `Stmt::While{…}`, `Stmt::Break`, `Stmt::Continue`; fill `collect_free_stmt`.
3. Parser: `parse_while` (incl. `do {} while(…);` and `while (var x = …)`), `break;`, `continue;`
   (+ tests). `while` cond requires parens (like `if`).
4. Checker: loop-depth guard for break/continue; while-let binds inner; fill casing/`in_stmts`/3
   rewrite passes (+ tests for the two new error codes + while-let binding).
5. Interpreter: `Signal::Break`/`Continue`; `Stmt::While` arm (pre/post cond + bind); break/continue
   arms.
6. Compiler: `loop_frames`; `compile_while`; `Stmt::Break`/`Continue` arms (pop-to-base + jump).
7. Transpiler: while / do-while / while-let / break / continue arms.
8. Differential `mutation_while_agrees` (counter, break, continue, do-while runs-once, while-let,
   nested-loop break-inner-only, two-binding observe). Guide example `guide/loops.phg`. Gate + commit.

## Step group B — C-style `for` (commit 2)

9. AST `Stmt::CFor`; parser disambiguates `for (Type x in …)` (existing) vs `for (init; cond; step)`
   by scanning for `in` vs `;` after the header opener. Backends + transpiler + checker arms.
10. Differential `mutation_cfor_agrees`; extend `guide/loops.phg`. Gate + commit.

## STATUS

**COMPLETE.** Both step groups landed in one green commit (C-`for` reuses the same loop machinery
as `while`, so the split was unnecessary). `while` / `do-while` / C-`for` / while-let / `break` /
`continue` all byte-identical run≡runvm≡real PHP 8.6; `break`/`continue` also retrofitted into the
existing for-in loop. No new `Op` (F5). 631 tests green, clippy+fmt clean.
